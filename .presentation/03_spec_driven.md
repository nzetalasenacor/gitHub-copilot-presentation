# Spec-Driven Development with Spec Kit

## Manual specs are fine when:

You already have a PRD process (Confluence, Notion, Google Docs)
Your team writes specs before coding anyway
You just want to reference docs from instruction files with #file:docs/prd.md
The project is small or you're the only developer

## Spec Kit adds value when:

Nobody writes specs until after the code is done (or never)
You want the agent to draft the spec from a short description — faster than writing it yourself
You need checkpoints enforced by the workflow, not by discipline
You want the spec, plan, and tasks in a standardized format every time
Multiple features need the same structure — Spec Kit templates guarantee consistency
You want the agent to trace every implementation decision back to a requirement
