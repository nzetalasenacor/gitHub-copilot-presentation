# Recipe API — Product Requirements

## Overview

A simple REST API that lets users create, browse, and search recipes.
Built with Java 21 and Spring Boot. Data stored in PostgreSQL.

## Users

- **Home cooks** who want to save and organize their recipes
- **API consumers** (mobile app, frontend) that need structured recipe data

## Features

### F1 — List recipes

As a user, I want to see all recipes so I can browse what's available.

- `GET /api/v1/recipes`
- Returns a paginated list (default 20 per page)
- Each item includes: id, title, category, prep time, created date
- Supports sorting by title or created date

### F2 — Get recipe details

As a user, I want to view a full recipe so I can cook it.

- `GET /api/v1/recipes/{id}`
- Returns: title, description, category, prep time, servings, ingredients list, steps list
- Returns 404 if recipe not found

### F3 — Create a recipe

As a user, I want to add a new recipe so I can save it for later.

- `POST /api/v1/recipes`
- Required fields: title, ingredients (at least 1), steps (at least 1)
- Optional fields: description, category, prep time in minutes, servings
- Returns 201 with the created recipe
- Returns 400 if validation fails

### F4 — Update a recipe

As a user, I want to edit a recipe so I can fix mistakes or improve it.

- `PUT /api/v1/recipes/{id}`
- Same validation rules as create
- Returns 200 with the updated recipe
- Returns 404 if recipe not found

### F5 — Delete a recipe

As a user, I want to remove a recipe I no longer need.

- `DELETE /api/v1/recipes/{id}`
- Returns 204 on success
- Returns 404 if recipe not found

### F6 — Search recipes

As a user, I want to search recipes by keyword so I can find what I'm looking for.

- `GET /api/v1/recipes/search?q={keyword}`
- Searches in title and description
- Returns paginated results, same format as F1
- Returns empty list if nothing matches (not 404)

## Data model

### Recipe

| Field       | Type         | Required | Notes                          |
|-------------|--------------|----------|--------------------------------|
| id          | Long         | auto     | Generated by database          |
| title       | String       | yes      | Max 200 characters             |
| description | String       | no       | Max 2000 characters            |
| category    | String       | no       | e.g. "breakfast", "dessert"    |
| prepTime    | Integer      | no       | In minutes                     |
| servings    | Integer      | no       | Number of servings             |
| ingredients | List<String> | yes      | At least 1 item                |
| steps       | List<String> | yes      | At least 1 item, ordered       |
| createdAt   | DateTime     | auto     | Set on creation                |
| updatedAt   | DateTime     | auto     | Set on every update            |

## API rules

- All responses use a standard envelope: `{ "data": ..., "error": null }`
- Error responses use: `{ "data": null, "error": { "code": "...", "message": "..." } }`
- All endpoints return `application/json`
- Input validation errors return 400 with specific field messages
- Internal errors return 500 with a generic message (never expose stack traces)

## Out of scope

- Authentication and authorization (future version)
- Image upload for recipes
- Rating or commenting on recipes
- User accounts
